# Understudy demo — 2 minutes 50 seconds

1. **0:00–0:20 — The premise.** Show the Practice Library and say: “Understudy drops me before a real change in a project's history, then asks me to rebuild the behavior in my own working copy.” Choose **Optimistic completion with rollback**.
2. **0:20–0:45 — Plan before code.** Answer the three planning questions. Show the coach checkpoint and say: “This is a direction check, not a solution. I can revise once, or use this plan and open the isolated working copy.” Click **Use this plan & open working copy**.
3. **0:45–1:10 — Work locally.** Copy the working-copy path, open it in the editor, and make the intentionally naive optimistic change. Point out that the fixture is a headless TypeScript data layer, so the tests—not a pretend browser UI—are the source of truth.
4. **1:10–1:35 — The amber signal.** Run checks. The normal suite is green while the behavioral check turns amber: “This is not a setup error. The familiar path works, and the edge case tells me what my first attempt missed.”
5. **1:35–1:58 — Guided independence.** Reveal one concept-level hint, make the rollback correction, and run checks again. If the API key is available, send one short question to the Coach card; show the **GPT-5.6** source chip. Without a key, show the honest disabled coach state while hints remain available.
6. **1:58–2:25 — Explain the evidence.** Show both checks passing, answer the explain-back question, and open the Mastery Report. Point to the separate engineering-outcome and independence signals, the timeline turning point, the diff summary, and the “coach messages” row.
7. **2:25–2:40 — Learn from the real change.** Scroll to **How it actually landed**. Explain: “Different shape, same behavior is fine. The tests decide.” The reference comparison is revealed only after completion; linked-project learners already know a commit SHA they selected themselves.
8. **2:40–2:50 — Build Week disclosure.** Voice over: “Codex accelerated the implementation and test authoring. GPT-5.6 powers structured server-side coaching and private Forge validation; deterministic tests remain authoritative.”

Record a public voiced video. Keep the amber signal and the plan-confirm checkpoint visible; they are the product's clearest teaching moments.
