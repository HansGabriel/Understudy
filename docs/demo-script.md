# Understudy demo - 2 minutes 40 seconds

1. **0:00-0:15 - The premise.** Start on Kata Lab to show the four preloaded easy-to-medium replays. Switch to **task-manager** and choose **Optimistic completion with rollback**: "Understudy drops me before a real change in a project's history, then asks me to rebuild the behavior in my own working copy."
2. **0:15-0:35 - Plan before code.** Answer the three planning questions. Show the coach checkpoint: "This is a direction check, not a solution. I can revise once, or use this plan and open the isolated working copy."
3. **0:35-0:50 - Work locally.** Open the working copy in the editor and make the intentionally naive optimistic change. Keep this beat short; the next test result is the evidence.
4. **0:50-1:15 - The amber signal.** Run checks. The project's own tests are green while the edge-case check turns amber: "This is not a setup error. The familiar path works, and the edge case tells me what my first attempt missed."
5. **1:15-1:38 - Guided independence.** Reveal one concept-level hint, make the rollback correction, and run checks again. If the API key is available, send one short Coach question and show the **GPT-5.6** source chip. Without a key, show the honest authored state while hints remain available.
6. **1:38-2:03 - Explain the evidence.** Show both checks passing, answer the explain-back question, and open the Mastery Report. Hold for three seconds on the **Session story** card and say: "The tests decided, not the AI." Then point to engineering outcome, independence, and the timeline turning point.
7. **2:03-2:25 - Learn from the real change.** Scroll to **How it actually landed**. Explain: "Different shape, same behavior is fine. The tests decide." The reference comparison is revealed only after completion.
8. **2:25-2:40 - Build Week disclosure.** Voice over: "Codex accelerated the implementation and test authoring. GPT-5.6 powers structured server-side coaching and private Forge validation; deterministic tests remain authoritative."

Record a public voiced video. Keep the Kata Lab selector, amber signal, plan-confirm checkpoint, and report summary visible; together they make the teaching loop clear without risking the three-minute limit.
