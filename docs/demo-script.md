# Understudy demo — 2 minutes 40 seconds

1. **0:00–0:20 — Problem.** Show the Practice Library and explain that ordinary tests often miss failure paths. Choose “Optimistic completion with rollback.”
2. **0:20–0:45 — Learner agency.** Submit the three-question plan. Note that GPT-5.6 responds as a coach, not a code generator, and the learner works in an isolated local worktree.
3. **0:45–1:15 — First attempt.** Open the path in the editor, make the intentionally naive optimistic change, and run checks.
4. **1:15–1:45 — Hero moment.** Highlight that the normal suite is green while the behavioral test turns amber: “This isn’t a bug in your setup — it’s the signal.” State that deterministic tests are authoritative.
5. **1:45–2:05 — Guided independence.** Reveal only level-one hint; it points to the concept without revealing code. Make the rollback correction and re-run checks.
6. **2:05–2:30 — Evidence.** Show all checks green, complete the explain-back, then open the report’s two separate signals and git-log timeline.
7. **2:30–2:40 — Build Week disclosure.** Voice over: Codex accelerated implementation and fixture/test authoring; GPT-5.6 powers server-only structured coaching and never sees the reference patch or learner code.
